# 解决的问题

# 在组件之间复用状态逻辑很难

React 没有提供将可复用性行为“附加”到组件的途径（例如，把组件连接到 store）。如果你使用过 React 一段时间，你也许会熟悉一些解决此类问题的方案，比如 render props 和 高阶组件。但是这类方案需要重新组织你的组件结构，这可能会很麻烦，使你的代码难以理解。如果你在 React DevTools 中观察过 React 应用，你会发现由 providers，consumers，高阶组件，render props 等其他抽象层组成的组件会形成“嵌套地狱”。尽管我们可以在 DevTools 过滤掉它们，但这说明了一个更深层次的问题：React 需要为共享状态逻辑提供更好的原生途径。

你可以使用 Hook 从组件中提取状态逻辑，使得这些逻辑可以单独测试并复用。Hook 使你在无需修改组件结构的情况下复用状态逻辑。 这使得在组件间或社区内共享 Hook 变得更便捷。

# 复杂组件变得难以理解

我们经常维护一些组件，组件起初很简单，但是逐渐会被状态逻辑和副作用充斥。每个生命周期常常包含一些不相关的逻辑。例如，组件常常在 componentDidMount 和 componentDidUpdate 中获取数据。但是，同一个 componentDidMount 中可能也包含很多其它的逻辑，如设置事件监听，而之后需在 componentWillUnmount 中清除。相互关联且需要对照修改的代码被进行了拆分，而完全不相关的代码却在同一个方法中组合在一起。如此很容易产生 bug，并且导致逻辑不一致。

在多数情况下，不可能将组件拆分为更小的粒度，因为状态逻辑无处不在。这也给测试带来了一定挑战。同时，这也是很多人将 React 与状态管理库结合使用的原因之一。但是，这往往会引入了很多抽象概念，需要你在不同的文件之间来回切换，使得复用变得更加困难。

为了解决这个问题，Hook 将组件中相互关联的部分拆分成更小的函数（比如设置订阅或请求数据），而并非强制按照生命周期划分。你还可以使用 reducer 来管理组件的内部状态，使其更加可预测。

# 难以理解的 class

除了代码复用和代码管理会遇到困难外，我们还发现 class 是学习 React 的一大屏障。你必须去理解 JavaScript 中 this 的工作方式，这与其他语言存在巨大差异。还不能忘记绑定事件处理器。没有稳定的语法提案，这些代码非常冗余。大家可以很好地理解 props，state 和自顶向下的数据流，但对 class 却一筹莫展。即便在有经验的 React 开发者之间，对于函数组件与 class 组件的差异也存在分歧，甚至还要区分两种组件的使用场景。

另外，React 已经发布五年了，我们希望它能在下一个五年也与时俱进。就像 Svelte，Angular，Glimmer等其它的库展示的那样，组件预编译会带来巨大的潜力。尤其是在它不局限于模板的时候。最近，我们一直在使用 Prepack 来试验 component folding，也取得了初步成效。但是我们发现使用 class 组件会无意中鼓励开发者使用一些让优化措施无效的方案。class 也给目前的工具带来了一些问题。例如，class 不能很好的压缩，并且会使热重载出现不稳定的情况。因此，我们想提供一个使代码更易于优化的 API。

为了解决这些问题，Hook 使你在非 class 的情况下可以使用更多的 React 特性。 从概念上讲，React 组件一直更像是函数。而 Hook 则拥抱了函数，同时也没有牺牲 React 的精神原则。Hook 提供了问题的解决方案，无需学习复杂的函数式或响应式编程技术。

# 函数式更新

这里有个建议setCount(x => x + 1)这个里面写成函数的形式，就和以前react中的setState一样

  // 以前的写法
  this.setState({
    x: 1
  })
  // 后来的写法
  this.setState(preState => {
    return preState + 1
  })

这里的setTimeout里面的setNum 只有用函数式的写法，取到的才是最新的值。 用这样的setNum({ number: num.number + 1 })写法，number的值是点击事件触发时的值。

import React, { useState } from 'react'

export default () => {
  let [num, setNum] = useState({ number: 0 })
  const add = () => {
    setNum({ number: num.number + 1 })
  }
  function showState() {
    setTimeout(() => {
      setNum(lastNum => ({ number: lastNum.number + 1 })) // 函数初始化
    }, 3000)
  }

  return (
    <div>
      <p>{num.number}</p>
      <button onClick={add}>+</button>
      <button onClick={showState}>showState</button>
    </div>
  )
}

# 单次渲染

多个useState，不会触发多次渲染，只会走一次render

import React, {useState} from 'react'

export default () => {
  const [count, setCount] = useState(0)
  const [count1, setCount1] = useState(0)

    <p>{count}</p>
    <p>{count1}</p>

      setCount(x => x+1)
      setCount1(x => x+1)
    }}>+1</button>
  </div>
}
  1. 每次渲染都有自己的state和props
  2. 每次渲染都有自己的事件处理函数

# 惰性初始化

初始化的时候，useState(() => sum())这样写，sum()只会执行一次。但是如果写成useState(sum())，每次render的时候sum()都会执行。 此外，这个sum()返回的默认值，只有第一次执行的时候有用，后面执行的时候不会再读默认值。

import React, { useState } from 'react'

function sum() {
  console.log('sum')
  return 1 + 2
}

export default () => {
  let [count, setCount] = useState(() => sum())
  // let [count, setCount] = useState(sum())
  const add = () => {
    setCount(preState => preState + 1)
  }

  return (
    <div>
      <p>{count}</p>
      <button onClick={add}>+</button>
    </div>
  )
}

# 错误一

Too many re-renders. React limits the number of renders to prevent an infinite loop.

import React, {useState, useEffect} from 'react'

const Greetings = () => {
  return <p>Hello</p>
}

export default () => {
  const [count, setCount] = useState(0)
  const [display, setDisplay] = useState(false)

  // 错误:递归
  if(count > 1) {
    setDisplay(x => true)
  }

  useEffect( () => {
    if(count > 1) {
      setDisplay(x => true)
    }
  }, [count > 1])

  return <div>
    <p>{count}</p>
    <button onClick={() => setCount(x => x + 1)}>+1</button>
    {display && <Greetings />}
  </div>
}

这个是造成无限递归，当count大于1的时候，会走render，然后会执行setDisplay，然后又要走render，再执行setDisplay，造成无限递归

# 错误二

Rendered more hooks than during the previous render.

import React, {useState, useEffect} from 'react'

const Greetings = () => {
  return <p>Hello</p>
}

export default () => {
  const [count, setCount] = useState(0)
  const [display, setDisplay] = useState(false)

  // Rendered more hooks than during the previous render.
  if (count > 1) {
    useEffect(() => {
      setDisplay(x => true)
    }, [])
  }

  useEffect( () => {
    if(count > 1) {
      setDisplay(x => true)
    }
  }, [count > 1])

  return <div>
    <p>{count}</p>
    <button onClick={() => setCount(x => x + 1)}>+1</button>
    {display && <Greetings />}
  </div>
}

这样的写法，避免了无限递归，因为这里的useEffect只会执行一次。 但是当count <= 1 的时候，hooks的数量是2，当count > 1 的时候 hooks的数量是3 hooks 是根据hook的顺序来确定每次的值的变化；一旦改变了就会发生不可预期的错误。

# 错误三

Rendered more hooks than during the previous render.

import React, {useState, useEffect} from 'react'

const Greetings = () => {
  return <p>Hello</p>
}

export default () => {
  const [count, setCount] = useState(0)
  const [display, setDisplay] = useState(false)

  // Rendered more hooks than during the previous render.
  if (count > 1) {
    useEffect(() => {
      setDisplay(x => true)
    }, [])
  } else {
    useEffect(() => {
      setDisplay(x => false)
    }, [])
  }

  useEffect( () => {
    if(count > 1) {
      setDisplay(x => true)
    }
  }, [count > 1])

  return <div>
    <p>{count}</p>
    <button onClick={() => setCount(x => x + 1)}>+1</button>
    {display && <Greetings />}
  </div>
}

这样表面上看，hooks的数量是对应上了，但是对于react来说，它会认为，3号位置上的hooks已经执行过了，不会再执行hooks了，但是当count > 1 的时候，3号位置上的hooks又出现了，这就会出现错误。

import React, { useState, useEffect, useRef } from 'react'

# useRef

const ref = React.createRef() 每次都会返回一个新的对象 const ref = useRef() 每次都会返回同一个对象 ref 会有个默认值 {current: null}

# 基本用法

这个例子和上一个例子基本相同，就是用ref替换了props可以达到同样的效果

import React, { useState, useEffect, useRef } from 'react'

const Input = props => {
  const ref = useRef()
  useEffect(() => {
    console.log(`value changed${props.value}`)
    ref.current.value = props.value // 这个时候的current是真实的DOM元素
  }, [props.value])

  return <input ref={ref} />
}

export default props => {
  const [value, setValue] = useState('')
  return (
    <div>
      <Input value={value} />
      <button onClick={() => setValue(x => Math.random())}>Click Me</button>
    </div>
  )
}

# useImperativeHandle

useImperativeHandle 这个hooks，可以让父组件，只修改子组件规定能修改的值，这里是：focus

import React, {
  useRef,
  useEffect,
  forwardRef,
  useImperativeHandle
} from 'react'

const Input = (props, ref) => {
  const refInput = useRef()

  useImperativeHandle(
    ref,
    () => ({
      focus: () => {
        refInput.current.focus()
      },
      setValue:(newValue)=>{
         refInput.current.value = newValue
      }
    }),
    []
  )

  useEffect(() => {
    refInput.current.value = props.initialValue
  }, [])
  return <input onChange={e => props.onChange(e.target.value)} ref={refInput} />
}

const MInput = forwardRef(Input)

export default () => {
  const r1 = useRef()
  const r2 = useRef()

  return (
    <div>

        ref={r1}
        initialValue={100}

      />
      <MInput
        ref={r2}
        initialValue={'hello'}
        onChange={x => console.log('2:' + x)}
      />

            console.log(r1)

            r1.current.setValue('hello world')

# useImperativeHandle的实现

import React, { useRef, useEffect, forwardRef } from 'react'

// useImperativeHandle 的实现
function useImperativeHandle(parentRef, handler) {
  parentRef.current = handler()
}

const Input = (props, parentFef) => {
  const refInput = useRef()

  useImperativeHandle(
    parentFef,
    () => ({
      focus: () => {
        refInput.current.focus()
      },
      setValue: newValue => {
        refInput.current.value = newValue
      }
    }),
    []
  )

  useEffect(() => {
    refInput.current.value = props.initialValue
  }, [])
  return <input onChange={e => props.onChange(e.target.value)} ref={refInput} />
}

const MInput = forwardRef(Input)

export default () => {
  const r1 = useRef()
  const r2 = useRef()

  return (
    <div>
      <MInput
        ref={r1}
        initialValue={100}
        onChange={x => console.log('1:' + x)}
      />
      <MInput
        ref={r2}
        initialValue={'hello'}
        onChange={x => console.log('2:' + x)}
      />
      <p>
        <button
          onClick={() => {
            console.log(r1)
            r1.current.focus()
            r1.current.setValue('hello world')

# useLayoutEffect

  • 其函数签名与 useEffect 相同，但它会在所有的 DOM 变更之后同步调用 effect
  • 可以使用它来读取 DOM 布局并同步触发重渲染
  • 在浏览器执行绘制之前useLayoutEffect内部的更新计划将被同步刷新
  • 尽可能使用标准的 useEffect 以避免阻塞视图更新

useLayoutEffect 能保证在DOM渲染完之后立刻执行，下例中可以看到，useLayoutEffect永远比useEffect先执行。

import React, {useLayoutEffect, useEffect, useState} from 'react'

export default () => {

  const [count, setCount] = useState(0)

  useEffect( () =>{
    console.log(`effect:` + new Date().getTime())
  }, [count])

  useLayoutEffect( () => {
    console.log(`layout:` + new Date().getTime())
  }, [count])

  useEffect( () => {
    const I = setInterval( () => {
      setCount(x => x + 1)
    }, 1000)

    return ()=> {
      clearInterval(I)
    }
  }, [])

  return <div>{count}</div>
}

# useCallback

useCallback本意是对函数进行缓存 useCallback,相当于缓存了数据，在useCallback中的count和外面的count，是两个值。useCallback这里是个闭包，所以才会导致这种结果 useCallback 第2个参数是依赖的变量的数组 当数组中变量发生改变后，就会重新生成新的函数

import React, { useState, useCallback } from 'react'

export default () => {
  const [count, setCount] = useState(0)

  const inc = useCallback(() => {
    console.log('count', count) // count 0
    setCount(count + 1)
  }, [])// useCallback
  console.log('render', count) // render 1

  return (
    <div>
      <p>点击次数：{count}</p>
      <button onClick={inc}>提交</button>
    </div>
  )
}

这里面两个值，始终是不变的。 换成下面这种写法，就可以了，因为x的值，取的是useCallback内部的count值，所以，每次都能够+1

import React, { useState, useCallback } from 'react'

export default () => {

# useMemo

useMemo 和 memo 也属于组件优化的一种写法

# 基本用法

例如下例中，无论是触发button的点击事件，还是触发input的输入框修改事件，MyCounter和render都会打印出来，这不是我们想要的效果。我们希望的是，在onChange事件触发的时候，MyCounter这个组件不应该触发更新，因为这两个组件是没有关联关系的。这个时候就要用上useMemo 和 memo

import React, { useState, useCallback, useMemo, memo } from 'react'

function MyCounter({ onButtonClick, data }) {
  console.log('MyCounter render')
  return (
    <div>
      <p>{data.number}</p>
      <button onClick={onButtonClick}>+</button>
    </div>
  )
}

export default () => {
  const [count, setCount] = useState({ number: 0 })
  let [value, setValue] = useState('')
  console.log('render')
  let onButtonClick = useCallback(() => {
    setCount({ number: count.number + 1 })
  }, [count])
  let data = { number: count.number }
  return (
    <div>
      <input onChange={event => setValue(event.target.value)}></input>
      <MyCounter onButtonClick={onButtonClick} data={data}></MyCounter>
    </div>
  )
}

import React, { useState, useCallback, useMemo, memo } from 'react'

function MyCounter({ onButtonClick, data }) {
  console.log('MyCounter render')
  return (
    <div>
      <p>{data.number}</p>
      <button onClick={onButtonClick}>+</button>
    </div>
  )
}

let MemoMyCounter = memo(MyCounter) // 绑定组件

export default () => {
  const [count, setCount] = useState({ number: 0 })
  let [value, setValue] = useState('')
  console.log('render')
  let onButtonClick = useCallback(() => {
    setCount({ number: count.number + 1 })
  }, [count])
  let data = useMemo(() => ({ number: count.number }), [count]) // 因为每次render都会重新给data赋值，所以每次data都会改变，所以需要用useMemo来缓存这个data，这样就不会触发更新了，只要不修改count，MemoMyCounter就不会更新，这样就实现了缓存
  return (
    <div>
      <input onChange={event => setValue(event.target.value)}></input>
      <MemoMyCounter onButtonClick={onButtonClick} data={data}></MemoMyCounter>
    </div>
  )
}

# useReducer

  • useState 的替代方案。它接收一个形如 (state, action) => newState 的 reducer，并返回当前的 state 以及与其配套的 dispatch 方法
  • 在某些场景下，useReducer 会比 useState 更适用，例如 state 逻辑较复杂且包含多个子值，或者下一个 state 依赖于之前的 state 等
  • useState 是 useReducer 的语法糖

这个是照抄官方文档上的例子

import  React, {useReducer} from 'react'
const initialState = {count: 0}

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return {count: state.count + 1}
    case 'decrement':
      return {count: state.count - 1}
    default:
      throw new Error()
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState)
  return (
    <>
      Count: {state.count}
      <button onClick={() => dispatch({type: 'decrement'})}>-</button>
      <button onClick={() => dispatch({type: 'increment'})}>+</button>
    </>
  )
}

export default Counter

# useState的实现

import React, { useReducer } from 'react'

function useState(initialState) {
  function reducer(state, action) {
    return { ...action.payload }
  }
  let [state, dispatch] = useReducer(reducer, initialState)
  function setState(newState) {
    dispatch({ type: '', payload: newState })
  }
  return [state, setState]
}

function Counter() {
  const [count, setCount] = useState({ number: 0 })
  return (
    <>
      Count: {count.number}
      <button onClick={() => setCount({ number: count.number + 1 })}>+</button>
      <button onClick={() => setCount({ number: count.number - 1 })}>-</button>
    </>
  )
}

export default Counter

# 自定义hooks

  • 有时候我们会想要在组件之间重用一些状态逻辑
  • 自定义 Hook 可以让你在不增加组件的情况下达到同样的目的
  • Hook 是一种复用状态逻辑的方式，它不复用 state 本身
  • 事实上 Hook 的每次调用都有一个完全独立的 state
  • 自定义 Hook 更像是一种约定，而不是一种功能。如果函数的名字以 use 开头，并且调用了其他的 Hook，则就称其为一个自定义 Hook，且必须要用use开头

# 自定义计数器

两个函数组件，虽然都是用的useNumber 这个hooks，但是自定义hooks，复用的是逻辑，而不是状态，所以，两个组件的值并不相等。

import React, { useState, useEffect } from 'react'

function useNumber(){
  const [number,setNumber] = useState(0);
  useEffect(() => {
     console.log('开启一个新的定时器')
     const $timer = setInterval(()=>{
      setNumber(number+ Math.random());
     },1000);
     return ()=>{
      console.log('销毁老的定时器')
         clearInterval($timer);
     }
  });
  return number;
}
function Counter1(){
  let number1 = useNumber();
  return (
      <>
          <p>{number1}</p>
      </>
  )
}
function Counter2(){
  let number = useNumber();
  return (
      <>
          <p>{number}</p>
      </>
  )
}
function App(){
  return <><Counter1/><Counter2/></>
}

# logger

核心就是useLogger这个方法，就是将useReducer重新包了一层。

import React, { useEffect, useState, useReducer } from 'react';

const initialState = 0;

function reducer(state, action) {
    switch (action.type) {
        case 'increment':
            return { number: state.number + 1 };
        case 'decrement':
            return { number: state.number - 1 };
        default:
            throw new Error();
    }
}

function init(initialState) {
    return { number: initialState };
}

function useLogger(reducer, initialState, init) {
    const [state, dispatch] = useReducer(reducer, initialState, init);
    let dispatchWithLogger = (action) => {
        console.log('老状态', state);
        dispatch(action);
    }
    useEffect(() => {
        console.log('新状态', state);
    }, [state]);
    return [state, dispatchWithLogger];
}

export default () => {
  const [state, dispatch] = useLogger(reducer, initialState, init)
  return (
    <>
      Count: {state.number}
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>

# promise

promise 有两种写法，分别注释了写法一和写法二 下面的代码usePromise这个方法依旧是重点。

import React, { useReducer } from 'react'

const initialState = 0

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { number: state.number + 1 }
    case 'decrement':
      return { number: state.number - 1 }
    default:
      throw new Error()
  }
}

function init(initialState) {
  return { number: initialState }
}

function usePromise(reducer, initialState, init) {
  const [state, dispatch] = useReducer(reducer, initialState, init)
  let dispatchPromise = action => {
    if (action.then && typeof action.then === 'function') {
      action.then(result => {
        dispatch(result)
      })
    } else if (
      action.payload &&
      action.payload.then &&
      typeof action.payload.then === 'function'
    ) {
      action.payload.then(payload => dispatch({ ...action, payload }))
    } else {
      dispatch(action)
    }
  }
  return [state, dispatchPromise]
}

export default () => {
  const [state, dispatch] = usePromise(reducer, initialState, init)

  return (
    <>
      Count: {state.number}
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      {/* 写法一 */}
      <button
        onClick={() => {
          dispatch(
            new Promise(resolve => {
              setTimeout(() => {
                resolve({ type: 'increment' })
              }, 1000)
            })
          )
        }}
      >
        promise+1
      </button>
      {/* 写法二 */}
      <button
        onClick={() =>
          dispatch({
            type: 'increment',
            payload: new Promise(resolve => {
              setTimeout(resolve, 1000)
            })
          })
        }
      >
        promise+1
      </button>
    </>
  )
}

# thunk

thunk 就是 dispatch 一个异步的函数。和redux-thunk是类似的操作

import React, { useEffect, useState, useReducer } from 'react'

const initialState = 0

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { number: state.number + 1 }
    case 'decrement':
      return { number: state.number - 1 }
    default:
      throw new Error()
  }
}

function init(initialState) {
  return { number: initialState }
}

function useThunk(reducer, initialState, init) {
  const [state, dispatch] = useReducer(reducer, initialState, init)
  let dispatchThunk = action => {
    if (typeof action === 'function') {
      action(dispatchThunk, () => state)
    } else {
      dispatch(action)
    }
  }
  return [state, dispatchThunk]
}

    <>
      Count: {state.number}
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button
        onClick={() =>
          dispatch((dispatch, getState) => {
            setTimeout(() => {
              console.log(getState())
              dispatch({ type: 'increment' })
            }, 1000)
          })
        }
      >
        delay
      </button>
    </>
  )
}

import React, { useState, useEffect, useLayoutEffect } from 'react';
import ReactDOM from 'react-dom';
function useRequest(url) {
    let limit = 5;
    let [offset, setOffset] = useState(0);
    let [data, setData] = useState([]);
    function loadMore() {
        setData(null);
        fetch(`${url}?offset=${offset}&limit=${limit}`)

            .then(pageData => {

                setOffset(offset + pageData.length);

}

import React, { useState, useEffect } from 'react'

function useRequest(url) {
  let limit = 5
  let [offset, setOffset] = useState(0)
  let [data, setData] = useState([])
  let [loading, setLoading] = useState(false)

  useEffect(() => {
    async function fetchData() {
      setData(null) // 为了出现加载中……的效果
      let pageData = await fetch(
        `${url}?offset=${offset}&limit=${limit}`
      ).then(response => response.json())
      setData([...data, ...pageData])
      setOffset(offset + pageData.length)
    }
    fetchData()
  }, [loading]) // Or [] if effect doesn't need props or state

  function loadMore() {
    setLoading(!loading)
  }
  return [data, loadMore]
}

import React, {useState, useEffect} from 'react'
import './animation.css'

function useAnimation(_clsName, duration = 300) {


  const [phase, setPhase] = useState(0)
  const [className, setClassName] = useState(_clsName)


  function play() {
    setPhase(1)
  }

  useEffect(() =>{
    if(phase === 1) {
      setClassName(_clsName + ' ' + className + '-start')
      setTimeout(() => {
        setClassName(_clsName + ' ' + className + '-end')
        setTimeout(() => {
          setClassName(_clsName)
          setPhase(0)
        })
      }, duration)
    } 
  }, [phase])


  return [className, play]

}

export default () => {
  const [className, playAnimation] = useAnimation('box')
  return <div>
    <div className={className}>
    </div>
  <button onClick={playAnimation}>play</button>
  </div>
}

import React, { useState, useEffect, useLayoutEffect } from 'react'
import './index.css'

function useMove(initialClassName) {
  const [className, setClassName] = useState(initialClassName)
  const [state, setState] = useState('')

  function bigger() {
    setState('bigger')
  }

  function smaller() {
    setState('smaller')
  }

  useEffect(() => {
    if (state === 'bigger') {
      setClassName(`${initialClassName} ${initialClassName}-bigger`)
    } else if (state === 'smaller') {
      setClassName(`${initialClassName} ${initialClassName}-smaller`)
    }
  }, [state])

  return [className, bigger, smaller]
}

  return (
    <div>
      <button onClick={bigger}>bigger</button>
      <button onClick={smaller}>smaller</button>
      <div className={className}></div>
    </div>
  )
}
