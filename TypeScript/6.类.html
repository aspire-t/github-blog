# 类

对于传统的 JavaScript 程序我们会使用函数和基于原型的继承来创建可重用的组件，但对于熟悉使用面向对象方式的程序员使用这些语法就有些棘手，因为他们用的是基于类的继承并且对象是由类构建出来的。 从 ECMAScript 2015，也就是 ES6 开始， JavaScript 程序员将能够使用基于类的面向对象的方式。 使用 TypeScript，我们允许开发者现在就使用这些特性，并且编译后的 JavaScript 可以在所有主流浏览器和平台上运行，而不需要等到下个 JavaScript 版本。

# ts中定义类

class Person {
  name: string; // 属性  省略了public关键词
  constructor(n: string) { // 构造函数实例化类的时候触发的方法
    this.name = n;
  }

  run():void{
    console.log(this.name)
  }

  getName(): string{
    return this.name;
  }
  setName(name: string): void {
    this.name = name;
  }
}

let p = new Person('张三');
p.run();

console.log(p.getName()) // 张三
p.setName('李四');
console.log(p.getName()) // 李四

这种在类里面，有初始化值的写法，有一种简化写法

class Person {
  // 传统写法
  // public name: string
  // constructor(name: string) {
  //   this.name = name
  // }
  // 简化写法
  constructor(public name: string) {}
}

const person = new Person('zhangsan')
console.log(person.name)

# getter和setter的使用

  • 如果在构造函数中，有私有属性，一般前面加_来表示
  • 用get和set可以暴露类中的私有属性，以便在类的外部使用
  • 在外部使用get和set暴露出来的方法的时候，不需要在方法后面加()，直接使用就可以。
class Person {
  constructor(private _name: string) {}
  get name() {
    return this._name + ' san'
  }
  set name(name: string) {
    const realName = name.split(' ')[0]
    this._name = realName
  }
}
const person = new Person('zhang')
console.log(person.name)
person.name = 'li si'
console.log(person.name)

# 单例模式

单例模式：不希望通过new Demo()生成多个Demo实例，只希望生成一个实例。 所以把constructor 用 private 声明，这样就不能通过new 来生成实例。 static 表示把 getInstance 这个方法，挂载在类上面，而不是挂载在类的实例上。

class Demo {
  private static instance: Demo
  private constructor(public name: string) {}
  static getInstance(name: string) {
    if (!this.instance) {
      this.instance = new Demo(name)
    }
    return this.instance
  }
}

const demo1 = Demo.getInstance('zhang san')
const demo2 = Demo.getInstance('li si')// 当实例已经创建了，再传入值，无效
console.log(demo2.name) // shang san
console.log(demo2 === demo1) // true

# ts中实现继承

  • 如果子类有自己的构造函数，则子类需要在自己的构造函数中，调用super()，来继承父类。
  • 如果父类有构造函数，并且有初始值，则子类在继承的父类的构造函数的时候，也需要传入一个初始值。
  • 如果没有构造函数，则可以在自己的方法中使用super调用父类的属性和方法。
class Person {
  name: string; // 属性  省略了public关键词
  constructor(n: string) { // 构造函数实例化类的时候触发的方法
    this.name = n;
  }

  run():string { 
    return `${this.name} is running`
  }
}

class Web extends Person {
  constructor(name:string){
    super(name); // 初始化父类的构造函数
  }
}

let w = new Web('李四');
w.run();

下面我们来看个更加复杂的例子。 super可以调用父类的公共方法和公共属性

class Animal {
  name: string
  constructor(name: string) {
    this.name = name
  }
  move(distance: number = 0) {
    console.log(`${this.name} moved ${distance}m.`)
  }
}

class Snake extends Animal {
  constructor(name: string) {
    super(name)
  }
  // 设置distance默认值
  move(distance: number = 5) {
    console.log('Slithering...')
    // 调用父类的move方法
    super.move(distance)
  }
}

class Horse extends Animal {
  constructor(name: string) {
    super(name)
  }
  move(distance: number = 45) {
    console.log('Galloping...')
    super.move(distance)
  }
}

let sam = new Snake('Sammy')
let tom: Animal = new Horse('Tommy')

sam.move()
tom.move(34)

# 类里面的修饰符

typescript 里面定义属性的时候给我们提供了 三种修饰符

public: 公有类型 在类里面、子类、类外部都可以访问

protected: 保护类型 在类里面、子类里面都可以访问 类外部没法访问

private: 私有 在类里面可以访问 子类、类外部没法访问

# ts 中的静态方法

class Person {
  name: string;
  age: number = 20;

  static sex = '男'

  constructor(n: string) {
    this.name = n;
  }

  run():void{
    console.log(this.name)
  }

  static print() { // 静态方法
    console.log('print')
    // console.log('print' + this.age)  报错，静态方法中没法直接调用类里面的属性
    // 调用静态属性
    console.log('print' + Person.sex)
  }
}

Person.print();
// 调用静态属性
console.log(Person.sex);

多态：父类定义一个方法不去实现，让继承它的子类实现 每一个子类有不同的表现

多态属于继承

class Animal {
  name: string;
  constructor(name: string) {
    this.name = name;
  }

  eat() { // 具体吃什么不知道， 继承它的子类去实现，每一个子类的表现不一样
    console.log('吃的方法')
  }
}

class Dog extends Animal {
  construct(name: string) {
    super(name)
  }

  eat() {
    return this.name + '吃肉';
  }
}

class Cat extends Animal {
  construct(name: string) {
    super(name)
  }
  
  eat() {
    return this.name + '吃老鼠';
  }
}

# 抽象方法和抽象类

typescript 中的抽象类： 它是提供其他类继承的基类，不能直接被实例化。

用abstruct 关键字定义抽象类和抽象方法，抽象类中的抽象方法不包含具体实现并且必须在派生类中实现

abstruct 抽象方法只能放在抽象类里面

抽象类和抽象方法用来定义标准

以上例来说 Animal 这个类要求它的子类必须包含eat方法

// 定义抽象方法和抽象类
abstruct class Animal {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
  abstruct eat(): any;
}

// 错误的写法，抽象类没法实例化
// let a = new Animal();

class Dog extends Animal {

  // 抽象类的子类必须实现抽象类里面的抽象方法
  constructor(name: string) {
    this.name = name;
  }
  eat() {
    console.log(this.name + '吃肉')
  }
}

let d = new Dog('小狗');
d.eat();
